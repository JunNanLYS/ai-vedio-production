# AI视频生产管理系统 - 项目规则

## 项目概述

**项目名称**: AI视频生产管理系统 (ai-video-production)  
**版本**: 1.0.1  
**类型**: Electron桌面应用程序  
**主要功能**: 订单管理、工作流管理、资产管理、数据统计

## 技术栈规范

### 前端技术栈
- **框架**: Vue 3 + TypeScript
- **构建工具**: Vite
- **桌面框架**: Electron
- **UI组件库**: Shadcn Vue (基于 Radix Vue)
- **样式方案**: Tailwind CSS + tailwindcss-animate
- **动效库**: @vueuse/motion
- **路由**: Vue Router 4
- **状态管理**: 组合式 API + reactive/ref
- **工具库**: @vueuse/core, clsx, tailwind-merge

### 后端技术栈
- **语言**: Python 3.11+
- **Web框架**: FastAPI
- **ORM**: SQLModel
- **数据库**: SQLite
- **日志**: Loguru
- **服务器**: Uvicorn
- **包管理**: uv

### 构建与部署
- **前端构建**: electron-vite
- **后端打包**: Nuitka
- **安装包**: electron-builder

## 项目架构

### 整体架构
```
Electron (主进程)
    ├── 前端渲染进程 (Vue 3 + TypeScript)
    └── 后端子进程 (FastAPI + SQLModel)
```

**关键设计**:
- Electron作为主进程，负责窗口管理和进程调度
- FastAPI作为子进程运行，由Electron启动和管理
- 后端端口由系统动态分配，通过标准输出传递给前端
- 前后端通过HTTP API通信

## 开发规范

### 语言与注释
- **代码注释**: 使用中文注释
- **文档语言**: 中文
- **API文档**: 使用中文描述

### 前端开发规范

#### 组件开发
- 使用 `<script setup lang="ts">` 语法
- 组件命名使用 PascalCase
- Props 使用 TypeScript 接口定义
- 使用组合式 API (Composition API)

#### 样式规范
- 使用 Tailwind CSS 工具类
- 遵循 mobile-first 响应式设计
- 使用 class-variance-authority 管理组件变体
- 动画使用 @vueuse/motion 或 tailwindcss-animate

#### 状态管理
- 简单状态使用 ref/reactive
- 复杂状态可考虑使用 Pinia
- 跨组件状态通过 props/emit 传递

#### API调用
- 统一使用 services/api.ts 封装
- 所有API调用返回 Promise
- 错误处理使用 try-catch

### 后端开发规范

#### 代码规范
- 遵循 PEP 8 编码规范
- 使用类型注解 (Type Hints)
- 函数和类添加文档字符串

#### 数据库操作
- 使用 SQLModel 定义模型
- 使用 Session 进行数据库操作
- 使用依赖注入获取 Session

#### API设计
- RESTful API 设计风格
- 使用 Pydantic 模型验证请求和响应
- 统一的错误处理和日志记录

#### 日志规范
- 使用 loguru 记录日志
- 关键操作记录 info 级别日志
- 错误记录 error 级别日志
- 调试信息记录 debug 级别日志

### 数据库设计规范

#### 表命名
- 使用单数形式 (order, workflow, asset)
- 使用下划线命名法 (snake_case)

#### 字段设计
- 主键使用自增整数 id
- 时间字段使用 datetime
- 外键使用 `{table}_id` 格式
- 必填字段使用 `Field(description="...")`

#### 索引优化
- 外键字段自动创建索引
- 查询频繁的字段考虑添加索引

## 核心功能模块

### 1. 订单管理模块
**功能**: 管理视频生产订单

**数据模型**:
- Order: 订单信息（公司名、视频数量、单价、收入、利润、状态）

**API接口**:
- `GET /api/orders` - 获取订单列表（分页）
- `GET /api/orders/recent` - 获取最近订单
- `POST /api/orders` - 创建订单
- `PUT /api/orders/{id}` - 更新订单
- `DELETE /api/orders/{id}` - 删除订单

**前端页面**: OrdersView.vue

### 2. 工作流管理模块
**功能**: 管理视频生产工作流

**数据模型**:
- Workflow: 工作流定义（名称、描述、步骤列表）
- Product: 产品实例（关联订单和工作流步骤）

**API接口**:
- `GET /api/workflows` - 获取全局工作流模板
- `POST /api/workflows` - 创建工作流
- `PUT /api/workflows/{id}` - 更新工作流
- `DELETE /api/workflows/{id}` - 删除工作流
- `GET /api/orders/{id}/workflow` - 获取订单工作流
- `POST /api/orders/{id}/apply-workflow/{workflow_id}` - 应用工作流到订单
- `POST /api/orders/{id}/products` - 添加产品
- `POST /api/products/{id}/move` - 移动产品步骤

**前端页面**: WorkflowsView.vue

### 3. 资产管理模块
**功能**: 管理视频生产素材

**数据模型**:
- Project: 项目
- SubCategory: 子分类
- Asset: 资产（prompt/image/audio/video/document）

**API接口**:
- `GET /api/projects` - 获取项目列表
- `POST /api/projects` - 创建项目
- `GET /api/projects/current` - 获取当前项目
- `POST /api/projects/switch/{id}` - 切换项目
- `DELETE /api/projects/{id}` - 删除项目
- `POST /api/projects/load` - 加载已有项目
- `GET /api/assets` - 获取资产列表
- `POST /api/assets` - 创建资产
- `POST /api/assets/upload` - 上传资产文件
- `DELETE /api/assets/{id}` - 删除资产
- `GET /api/assets/{id}/preview` - 获取资产预览

**前端页面**: AssetsView.vue

### 4. 仪表盘模块
**功能**: 展示统计数据

**API接口**:
- `GET /api/dashboard/stats` - 获取统计数据

**前端页面**: DashboardView.vue

## 进程通信机制

### IPC通信
前端通过 preload 脚本与主进程通信：

**主要IPC接口**:
- `get-backend-port` - 获取后端端口
- `subscribe-backend-logs` - 订阅后端日志
- `open-directory` - 打开目录
- `open-file` - 打开文件
- `select-directory` - 选择目录
- `window-minimize` - 最小化窗口
- `window-maximize` - 最大化窗口
- `window-close` - 关闭窗口
- `check-for-updates` - 检查更新
- `download-update` - 下载更新
- `quit-and-install` - 安装更新

### 后端进程管理
- 开发环境: `uv run python main.py`
- 生产环境: Nuitka打包的可执行文件
- 端口分配: 系统自动分配空闲端口
- 进程监控: 监听 stdout/stderr，记录日志

## 自动更新机制

使用 electron-updater 实现自动更新：

1. 启动时检查更新（静默模式）
2. 发现更新通知用户
3. 用户确认后下载更新
4. 下载完成后提示安装
5. 用户确认后重启安装

## 扩展开发指南

### 添加新API
1. 在 models/ 创建数据模型
2. 在 routers/ 创建路由文件
3. 在 routers/__init__.py 导出路由
4. 在 main.py 注册路由
5. 在前端 services/ 创建API服务

### 添加新页面
1. 在 views/ 创建页面组件
2. 在 router/index.ts 添加路由
3. 在 Sidebar.vue 添加导航菜单

### 添加新组件
1. 在 components/ 创建组件目录
2. 使用 Shadcn Vue 组件
3. 如需新UI组件: `npx shadcn-vue@latest add [component]`

## 常见问题

### 后端启动失败
- 检查 Python 版本 (需要 3.11+)
- 检查依赖安装: `uv sync`
- 查看后端日志

### 前端无法连接后端
- 检查后端是否启动
- 检查端口是否正确
- 查看控制台错误

### 数据库问题
- 数据库位置: 用户数据目录（生产）/ 项目根目录（开发）
- 迁移脚本: database.py

### 构建失败
- 检查 Node.js 版本
- 检查 Python 环境
- 检查构建依赖
